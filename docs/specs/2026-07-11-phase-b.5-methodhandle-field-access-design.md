## 0. 为什么必须做 MethodHandle(Step 0 源码核验)

`Method.invoke`/`Constructor.newInstance` 走 `useNativeAccessor`(`!VM.isJavaLangInvokeInited()`
→ true,rustj 不跑 initPhase3)→ `NativeAccessor.invoke0`/`newInstance0` native 捷径(已移植,
`c9a6bc1`)。**字段无此捷径**:

`jdk/internal/reflect/MethodHandleAccessorFactory.java:172-210` `newFieldAccessor`:
```java
if (!VM.isJavaLangInvokeInited()) {                      // :173
    throw new InternalError(... "cannot be accessed reflectively before java.lang.invoke is initialized");
}
...
var getter = JLIA.unreflectField(field, false);          // :185
var setter = isReadOnly ? null : JLIA.unreflectField(field, true);  // :186
```

`jdk/internal/reflect/` 全树**无**字段访问 native(仅 `DirectMethodHandleAccessor.invoke0` 是
方法用,`ConstantPool.java` 是 CP 用,`Field.getTypeAnnotationBytes0` 是注解用)。**字段访问
强制走 MethodHandle**,无 native 兜底。故 Field.get/set = MethodHandle 直接调用(顺延候选 g)。

## 1. 完整调用链(Java 侧,纯字节码 + 标注的 native)

```
Field.get(obj)                                    // Field.java:470
 → getFieldAccessor(obj).get(obj)                 // Field.java:1294 惰性建 accessor
 → ReflectionFactory.newFieldAccessor(field,ovr)  // ReflectionFactory.java:98
 → MethodHandleAccessorFactory.newFieldAccessor   // MethodHandleAccessorFactory.java:172
    ├─ VM.isJavaLangInvokeInited()  ← gate(:173)  须 true,否则 InternalError
    ├─ ensureClassInitialized(defc)               // 已支持(4.24 ensureClassInitialized0)
```

**链 C:解析 MemberName**(`checkField` → `MemberName.Factory.resolve`)

**链 D:`DirectMethodHandle.make`(DirectMethodHandle.java:113-124,字段分支)**
```java
LambdaForm lform = preparedFieldLambdaForm(member);        // :114  (rustj 不解释 LF)

    long offset = MethodHandleNatives.staticFieldOffset(member);  // native,:58 → :116
```

**链 E:accessor.get → `MethodHandle.invoke`(签名多态)→ `invokeBasic` → LambdaForm → Unsafe**
rustj **不解释 LambdaForm**。`MethodHandleXxxFieldAccessorImpl.get(obj)` 调
`getter.invoke(obj)`(签名多态法,JVMS §2.9),JVM 经 `linkToVirtual`/`invokeBasic` 进 VM。
rustj 在解释器拦截 `invokevirtual/invokestatic java.lang.invoke.MethodHandle.{invoke,invokeExact,invokeBasic}`
为**MH 调用钩子**,按 receiver 的 DMH 子类直读 member → 做字段访问(类似 indy bootstrap 短路)。

## 2. 关键设计决策:shortcut,不解释 LambdaForm

HotSpot 经 LambdaForm 解释/JIT 字段访问;rustj **直读 DMH.member**(MemberName 含
clazz/name/type/isStatic/refKind)→ 用既有 `instance_field`/`static_field` 做访问。故
`objectFieldOffset`/`staticFieldOffset`/`staticFieldBase` 的**返回值可被忽略**(dummy),钩子不
读 DMH 的 offset/base 字段,只读 member。这把"忠实但巨大的 LF 解释"降为"语义等价的直访"。

**理由**(对应 §1 链):MemberName 已含访问字段所需的全部信息(声明类+名字+类型+静态与否+
getter/setter via refKind)。offset/base 是 HotSpot 性能优化(直接内存偏移),非正确性所需。

## 4. 数据与字段映射(Rust 侧)

**MemberName 须可读的 Java 字段**(钩子用,经 `instance_field`/`static_field` 读 Instance):
- `clazz`(Class 镜像)→ `mirror_internal_name` 得内部名 → `registry.get`
- `name`(String)→ 池解码
- `type`(MethodType for 方法 / Class for 字段)→ 字段:Class 镜像 → 组件类型
- `flags`(int)→ `MethodHandleNatives` 常数(refKind 位、isStatic、isFinal…)经 `getNamedCon` 对齐,
  **或**在钩子里直接读 `MemberName.getReferenceKind()`/`isStatic()`(纯 Java 字节码法,经解释器跑)。

**钩子优先用 Java 字节码法**:`receiver.internalMemberName()` 返 member(纯 Java 法,DirectMethodHandle:182),
再 member.getReferenceKind()/getDeclaringClass()/getName()(纯 Java 法)。钩子把这些当普通
invokevirtual 跑 → 复用既有解释器,免重复实现常数表。仅当某法触发未绑 native 才补。

## 5. 复用既有能力

- 字段访问:`instance_field`/`static_field`(扁平布局,4.2)、`set_static_field`(launch)。
- 装箱/拆箱:`Integer`/`Long`/...(真 java.base,4.10t 系列);accessor 子类按类型分派已做拆装箱。
- 类初始化:`ensureClassInitialized0`(4.24)。
- 镜像:`intern_class_mirror`/`mirror_internal_name`(4.12)。
- 解释器复入:`interpret_with`(§6 NLL trick)。
- indy 短路先例:`LambdaMetafactory.metafactory`(4.10aa)—— 同模式(DMH 创建期短路 + 调用期拦截)。

## 6. 不做(范围控制)

- LambdaForm 解释(全部 DMH 表达式)、BoundMethodHandle、guardWithTest/combinator 链。
- `linkToVirtual`/`linkToStatic`/`linkToSpecial`/`linkToInterface` 的方法 DMH 调用(Method.invoke
  已有 native invoke0 捷径;MethodHandle 用户直调 `invokeExact` 跑 Stream 顺延 — 本期仅字段)。
- VarHandle(独立于 Field,顺延)。
- MethodHandle.invoke 对**方法** DMH(非字段)— 与 invoke0 重复,顺延。

## 7. 风险与回退

- **JLIA 安装**:`MethodHandleImpl` <clinit> 若触发未绑 native 或重初始化死锁 → 退路:在 Rust
  bootstrap 里手动 `new_instance(System$1 风格)` 一个最小 JavaLangInvokeAccess 占位 + 置
  SharedSecrets 字段(类比 4.30 install_java_lang_access)。先试自然 <clinit>,失败再退。
- **MemberName 解析常数**:`flags` 位定义可能版本错位(memory `jdk-master-source-vs-jmod-version-mismatch.md`)
  → 以**本机 jmod 实测**为准,源码常数仅参考;优先用 Java 字节码法读 refKind 规避常数表。
- **accessor 子类 <clinit>**:`MethodHandleXxxFieldAccessorImpl` 可能引用未就绪类 → 经 `load_closure`
  预载整个引用闭包(既有机制)。

## 8. 验收

- lib 单测:DMH 字段访问器直调 + isJavaLangInvokeInited 翻转 + MemberName 解析。
- javac 端到端闸门(类比 `tests/reflection_invoke.rs`):`Probe` 含 `int i; static String s; long j;`
  断 `f.get(p)`/`f.set(p,v)`/`f.get(null)`(static)各原语类型正确。
- 全套测试绿、clippy 净、零 unsafe、零依赖。

## 9. B.5.1 实现纪要(2026-07-12,GREEN)

`unreflectField` 返非 null DMH 链路打通(闸门 `tests/reflection_field_unreflect.rs`)。TDD
红→绿逐 native 推进,实际触发的 native 链(及对应绑定):

1. **`MethodHandleNatives.init(MemberName, Object)V`** —— 新子模块
   `native/java_lang_invoke.rs`(mod.rs 加 `java/lang/invoke/` 前缀路由,先于 `java/lang/`)。
   Field 分支移植 `init_field_MemberName`(methodHandles.cpp:365):读 `Field.clazz`/`modifiers`
   → 置 `MemberName.clazz` + `flags = mods | MN_IS_FIELD | (refKind<<24)`(refKind = isStatic ?
   REF_getStatic(2) : REF_getField(1))。`isResolved()` 自动 true(resolution=null)。
2. **`Object.clone()Object`** —— java_lang.rs。浅拷贝(Oop::clone → heap.alloc)。MemberName.clone
   调用(makePreparedFieldLambdaForm 触发)。
3. **`Class.{getEnclosingMethod0,getDeclaringClass0,getSimpleBinaryName0}`** —— 三 private native 均
   返 null(顶级类语义)。解锁 `MethodType.toString → Class.getSimpleName` 链(顶级类走"取类名剥包")。
4. **`MethodHandleNatives.resolve(MemberName,Class,int,Z)MemberName`** —— shortcut 原样返回入参
   (入参构造器已置 resolution=null=已解析;`-ea` off 故 vminfoIsConsistent/getMemberVMInfo 不触)。
5. **数组 Object 继承法虚分派** —— invoke.rs `dispatch_array_object_method`(clone/getClass/
   hashCode/equals;invokevirtual+invokeinterface 共用)。解锁 `LambdaForm$BasicType[].getClass()`
   等(LF 准备对数组调 Object 法)。
6. **`Array.newArray(Class,int)Object`** —— java_lang.rs `array_new_array`。组件镜像→内部名→
   (数组描述符,默认槽)造数组。解锁 `LambdaForm$BasicType.<clinit>`(Arrays.copyOf→Array.newInstance)。
7. **`objectFieldOffset`/`staticFieldOffset`/`staticFieldBase`** —— dummy(返 0/null);B.5.2 钩子
   直读 member,不读 offset/base(设计 §2)。
8. **`Unsafe.shouldBeInitialized0(Class)Z`** —— jdk_internal.rs。`init_state != Done`。解锁
   `DirectMethodHandle.shouldBeInitialized`(jmod 行号错位:trace 示 366 实为 shouldBeInitialized0)。

`expand`/`getMemberVMInfo`/`registerNatives` 未显式绑(registerNatives 全局 no-op;另两者仅经
`-ea` 断言触达,rustj 断言全关)。javaLangInvokeInited 引导经 `MethodHandleNatives.<clinit>`
自然翻转(VM.setJavaLangInvokeInited),`MethodHandleImpl.<clinit>` 装 JLIA —— `launch.rs
bootstrap_java_lang_invoke` 仅 `ensure_class_initialized` 二者。

**质量**:343 lib + 全集成绿、clippy 净、零 unsafe、零依赖。**下一层 B.5.2**:解释器
`MethodHandle.{invoke,invokeExact,invokeBasic}` 钩子(receiver 为字段 DMH → 读 member →
getfield/putfield/getstatic/putstatic)。
