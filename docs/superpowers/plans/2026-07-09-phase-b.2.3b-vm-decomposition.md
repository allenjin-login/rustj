# Phase B.2.3b 实现计划:Vm 分解 + Arc<VmShared> + 共享字段 Mutex

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `Vm` 的跨线程共享态收敛进 `Arc<VmShared>`(每字段 `Mutex`),把 ~35 个方法按职责拆入子模块文件,新增 `ThreadManager`——为 Phase B.3 真多线程(synchronized 互斥 / Thread.start0 / wait-notify)打地基,单线程行为保持全绿。

**Architecture:** `Vm<'a> { shared: Arc<VmShared<'a>>, thread: ThreadContext }`。`VmShared` 的可变共享字段全 `Mutex` 化(Heap/StringPool/monitors/next_tid/exception_meta/class_mirrors/mirror_class/module_mirrors/unnamed_module);`registry: Option<&'a ClassRegistry>` 仍是 `Copy` 的 `&'a`(§6 NLL 模式保留)。方法留在 `impl Vm`(子模块文件 `mirrors.rs`/`monitors.rs`/`threads.rs`/`exceptions.rs`),经 `self.shared.<field>.lock()` 访问共享态、`self.thread.<field>` 访问线程隔离态;`shared` 升 `pub(crate)`。`Vm::new(&'a ClassRegistry)` / `Default` 签名不变(构造点零改动);新增 `Vm::from_shared(Arc<VmShared>)`(`pub(crate)`,B.3b 派生线程用)。栈帧法下沉 `impl ThreadContext`。

**Tech Stack:** Rust edition 2024、`std::sync::{Arc, Mutex}`(零依赖、零 unsafe)、`#![deny(unsafe_code)]`。

## 关键模式(Commit 1 立此模式,后续复用)

### 模式 P1:inline 调用(Mutex 后无需改,Deref 自动)
```rust
vm.heap_mut().alloc(Oop::Instance(inst))   // heap_mut()->MutexGuard, DerefMut->&mut Heap, alloc 跑完 guard 即释
vm.heap().get(r)                            // 用于 match scrutinee 且臂内无 &mut vm 时,temp guard 跨 match 存活但无冲突
```

### 模式 P2:跨语句绑定(Mutex 后 E0716/E0502,须提取 owned)
```rust
// 改前:
let Some(Oop::Instance(i)) = vm.heap().get(exc) else { return false };  // i 借临时 guard → E0716
// 改后:
let oop = vm.heap().get(exc).cloned();        // Option<Oop> owned
let Some(Oop::Instance(i)) = oop else { return false };
```

### 模式 P3:match scrutinee 臂内需 &mut vm(须提取 owned)
```rust
// 改前:
match vm.heap().get(r) {                      // guard 跨整个 match 存活
    Some(Oop::Instance(i)) => { vm.heap_mut().alloc(..) }  // &mut vm 与活跃 guard 冲突 E0502
}
// 改后:
let oop = vm.heap().get(r).cloned();
match oop {
    Some(Oop::Instance(i)) => { vm.heap_mut().alloc(..) }  // 无 guard,OK
    _ => {}
}
```

### 模式 P4:内部法持锁后递归 &mut self(drop-before-recurse)
```rust
// intern_class_mirror:锁 → 查缓存 → drop 锁 → alloc(可 &mut self)→ 再锁 → 插入
fn intern_class_mirror(&mut self, name: &str) -> Reference {
    let cached = self.shared.class_mirrors.lock().unwrap().get(name).copied();
    if let Some(r) = cached { return r; }
    let r = self.alloc_class_mirror_instance();           // &mut self OK(无锁持有)
    {
        let mut g = self.shared.class_mirrors.lock().unwrap();
        g.insert(name.to_string(), r);
    }
    self.shared.mirror_class.lock().unwrap().insert(r, name.to_string());
    self.populate_class_mirror_fields(r, name);           // &mut self OK
    r
}
```

### 模式 P5:返引用 accessor 改 owned(Mutex 后不能返 &T 穿 guard)
```rust
// 改前:fn mirror_internal_name(&self, r: Reference) -> Option<&str>
// 改后:fn mirror_internal_name(&self, r: Reference) -> Option<String>
//        { self.shared.mirror_class.lock().unwrap().get(&r).cloned() }
// 调用方:.map(|s| s.as_str()…) 或 .as_deref() 适配。
```
