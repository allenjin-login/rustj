# Layer 4.14c — java.base Module.descriptor + exportedPackages 填充(解锁端到端反射)

> **状态**:设计(2026-07-11)。`/goal` 主线:解锁 Layer 4.15b 端到端 `Method.invoke`。
> 4.15b invoke0/newInstance0 native 已完成(commit `c9a6bc1`),但 `Method.invoke` 字节码路径的
> `checkAccess` 阻塞于 java.base Module.descriptor=null。本层填充之。

## 1. 目标

让 `Module.isExported("java.lang")`(及所有 java.base 导出包)在 rustj 返 true,使
`Method.invoke` 的 `checkAccess`→`verifyPublicMemberAccess`→`Module.isExported` 通过 →
**端到端 `Method.invoke` 绿**(移除 `tests/reflection_invoke.rs` 的 `#[ignore]`)。

## 2. Step 0 源码核验(决定性发现)

### 2.1 NPE 根因

`Method.invoke` happy-path(public/public/exported)经:

```
Method.invoke → checkAccess → verifyAccess → slowVerifyAccess
  → Reflection.verifyPublicMemberAccess(Reflection.java:238)
  → Module.isExported(Module.java:592) [1-arg, other=EVERYONE_MODULE]
  → Module.implIsExportedOrOpen(Module.java:705/751) → descriptor.isOpen() → NPE
```

java.base 是**命名**模块 → `isNamed()`=true → 无 fast-path;`descriptor`=null → NPE。

### 2.2 implIsExportedOrOpen 实际读什么(Module.java:741-764)

```java
private boolean implIsExportedOrOpen(String pn, Module other, boolean open) {
    if (!isNamed()) return true;                              // java.base 命名 → 不走
    if (other == this && descriptor.packages().contains(pn))  // other=EVERYONE≠this → 不走
        return true;
    if (descriptor.isOpen() || descriptor.isAutomatic())      // ★ descriptor 须非 null
        return descriptor.packages().contains(pn);            //   java.base 非 open/auto → false
    if (isExplicitlyExportedOrOpened(pn, other, open))        // ★ 真正判定处
        return true;
    if (isReflectivelyExportedOrOpened(pn, other, open))      // addExports
        return true;
    return false;
}
```

**关键**:`descriptor` 仅在 `isOpen()`/`isAutomatic()`(须=false)与未命中分支的 `packages()` 被读。
真正判定在 `isExplicitlyExportedOrOpened`(812-828):

```java
private boolean isExplicitlyExportedOrOpened(String pn, Module other, boolean open) {
    Map<String, Set<Module>> openPackages = this.openPackages;
    if (openPackages != null && allows(openPackages.get(pn), other)) return true;
    if (!open) {
        Map<String, Set<Module>> exportedPackages = this.exportedPackages;   // ★ Module 实例字段
        if (exportedPackages != null && allows(exportedPackages.get(pn), other)) return true;
    }
    return false;
}
```

⇒ **判定读 `this.exportedPackages`(Module 实例 Map<String,Set<Module>>),非 `descriptor.exports()`**。
`allows`(835)查 `targets.contains(EVERYONE_MODULE)`。

### 2.3 exportedPackages 的正常填充(Module.initExports,1470-1497)

```java
private static void initExports(Module m, Map<String, Module> nameToModule) {
    Map<String, Set<Module>> exportedPackages = new HashMap<>();
    for (Exports exports : m.getDescriptor().exports()) {
        String source = exports.source();
        if (exports.isQualified()) { /* 限定导出:addExports0 + targets set */ }
        else {                       // 非限定导出
            addExportsToAll0(m, source);                       // native(rustj 无 JVM 模块表 → no-op)
            exportedPackages.put(source, EVERYONE_SET);        // ★
        }
    }
    if (!exportedPackages.isEmpty()) m.exportedPackages = exportedPackages;
}
```

⇒ **非限定导出 → `{pkg → EVERYONE_SET}`**。`EVERYONE_SET = Set.of(EVERYONE_MODULE)`
(Module.java:433-434,<clinit> 置;`EVERYONE_MODULE = new Module(null)`)。

### 2.4 rustj 现状(数据缺口)

- `metadata::module::ModuleDescriptor`(Rust)已**完整解析** java.base 的 exports/packages
  (`ModuleExports{package, flags, to_modules}`,class_path.rs:78),但 `ClassPath::add`
  **只保留 `name`,丢弃 exports/packages**。
- 4.14a `intern_named_module`(mirrors.rs)创建 java.base Module 仅置 `name` 字段;
  `descriptor`/`exportedPackages` 均 null。
- `java.lang.module.ModuleDescriptor` 字段(`ModuleDescriptor.java:1268-1277`):`name`(String)、
  `open`/`automatic`(boolean,默认 false)、`exports`(Set<Exports>)、`packages`(Set<String>)等。
  `Exports`(`:422-424`):`source`(String)、`targets`(Set<String>,空=非限定)。

## 3. 架构

### 3.1 保留 Rust ModuleDescriptor(registry 数据层)

- `ClassRegistry` 增 `module_descriptors: Mutex<HashMap<String, ModuleDescriptor>>`(name→Rust 描述符)。
- `ClassPath::add` 返回/保留完整 Rust `ModuleDescriptor`(不只 name);`load_closure` 据 `module_name`
  把完整描述符登记进 registry(同 `set_class_module` 时机)。
- 访问器 `module_descriptor(name) -> Option<ModuleDescriptor>`(clone)。

### 3.2 bootstrap 填充(launch.rs)

新增 `populate_module_exports(vm, "java/base 模块镜像")`,在 `bootstrap_module_system` 末尾调:

1. **ensure `java/lang/Module` 初始化**(触发 `<clinit>` → `EVERYONE_MODULE`/`EVERYONE_SET` 创建)。
2. **读 `Module.EVERYONE_SET` 静态字段**(经 `resolve_static_field` + `static_storage` 读)。
3. **构造 java `HashMap` exportedPackages**:
   - `new_instance(HashMap)` + 对 java.base 每个非限定 export 调 `HashMap.put(pn_string, everyone_set)`
     (经 `interpret_with` 跑 `put(LObject;LObject;)Object`)。
4. **构造 java `ModuleDescriptor`**(最小忠实):`new_instance` + 设 `name`="java.base"
   (`open`/`automatic` 默认 false 即可——访问检查仅读这两布尔,不读 `exports()`/`packages()`)。
   - **faithfulness 注**:`exports()`/`packages()` 暂空(访问检查不读);若后续 Module.toString/其他
     读之,扩展为真 Exports/Set 构造(顺延)。
5. **写 java.base Module 实例字段**:`descriptor`(ord)+`exportedPackages`(ord),经
   `flattened_instance_fields` 查序号 + `heap_mut().get_mut().set_field`。

### 3.3 解锁链

`Method.invoke` → checkAccess → `Module.isExported("java.lang")`
→ implIsExportedOrOpen:isNamed true / other≠this / isOpen=false → isExplicitlyExportedOrOpened
→ `exportedPackages.get("java.lang")`=EVERYONE_SET → allows(含 EVERYONE_MODULE)→ **true**。

## 6. 顺延项

- **ModuleDescriptor.exports()/packages() 完整化**:当前仅设 name(访问检查够用);真 Exports/Set
  构造顺延(Module.toString / 完整模块反射需要时)。
- **限定导出(qualified exports)**:java.base 对 `jdk.unsupported` 等的限定导出;本层仅填非限定
  (EVERYONE_SET),限定导出顺延。
- **openPackages 填充**:同 exportedPackages;反射 `setAccessible` 深度反射路径需 open,
  顺延(当前 public 法不 setAccessible 即可调)。
- **`addExports0`/`addExportsToAll0` native**:rustj 无 JVM 模块表 → 恒 no-op(exportedPackages
  Map 是唯一真源)。
